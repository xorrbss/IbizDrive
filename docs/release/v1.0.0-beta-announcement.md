## 1. 베타 범위

**사내 베타 = 내부 사용자 전용**. 외부 출시는 v1.x 별도 트랙.

- 사내 도메인 + 사내 IdP/SSO 가정 (외부 인터넷 노출 없음)
- 데스크탑 브라우저 메인 (Chrome/Edge/Safari 최신)
- **모바일 미지원** — `CLAUDE.md §3 원칙 13` 명시, mobile-view 항목 폐기

## 2. 핵심 기능

### 2.1 파일/폴더 탐색
- 워크스페이스 prefix 라우팅 — `/d/:deptId/...` (부서), `/t/:teamId/...` (팀), `/shared/...` (공유함), `/trash/d/...` · `/trash/t/...` (휴지통)
- URL이 진실의 출처 — 사이드바 expand state만 localStorage 유지
- 폴더 트리 + breadcrumb + 가상화 list/grid view
- DnD 이동 (dnd-kit) + 업로드 DnD (window 네이티브) 분리
- 키보드 탐색 + 단축키 cheat sheet (`?`)

### 2.2 파일 관리
- 멀티 파일 업로드 (multipart, ADR #36) — quota 가드 (413 QUOTA_EXCEEDED)
- 버전 관리 (4탭 RightPanel: 상세 / 버전 / 활동 / 권한)
- 버전 다운로드 (RFC 5987 `Content-Disposition: attachment`)
- 버전 복원 (옵션 A — `current_version_id` 재지정)
- 휴지통 30일 보존 + 일괄 복원/영구삭제 (admin)

### 2.3 공유 / 권한
- 폴더/파일 공유 (subject = USER / DEPT / TEAM / everyone)
- 권한 grant 다이얼로그 (`GrantPermissionDialog`) — preset 5종 + subject 라디오
- 만료 cron — 공유/권한 자동 만료 + audit
- 감사 로그 (audit_log) — append-only DB-level 강제

### 2.4 검색
- 파일/폴더 통합 검색 (`/api/search`) — q + 워크스페이스 scope

### 2.5 관리자 콘솔
- `/admin/members` — 사용자 초대 / role 변경 / 비활성 / displayName 편집 / quota 한도 설정 (KPI 4장 + Filter Bar — design fidelity)
- `/admin/departments` — 부서 CRUD (생성/검색/rename/(de)activate)
- `/admin/teams` — 팀 관리 (생성/멤버/리더/archive/restore)
- `/admin/permissions` — 권한 매트릭스 viewer + 권한 부여 다이얼로그
- `/admin/sharing` — 공유 정책 (frontend mock — backend endpoint v1.x)
- `/admin/storage` — 시스템 용량 / 부서별 사용량 / 정리 기록
- `/admin/retention` — 휴지통 보존 일수 mutation
- `/admin/audit` — 감사 로그 viewer + CSV/JSON export + Severity tab
- `/admin/system` — cron 4종 ON/OFF (ADMIN-only)
- `/admin/trash/all` — 전역 휴지통 viewer + bulk 복원/영구삭제

### 2.6 디자인
- 디자인 시스템 토큰 (light/dark + 4 variants: linear/notion/dropbox/terminal) + density 3종 (compact/default/comfortable)
- 파일 타입 컬러 아이콘 10종 (doc/pdf/sheet/slides/image/video/figma/code/archive/folder)

## 4. 운영 안내

### 4.1 첫 로그인
1. 운영자가 보낸 초대 메일 확인 — 임시 비밀번호 포함
2. 사내 SSO 도메인 + 이메일 + 임시 PW로 로그인
3. `/account/password?force=1` 강제 비밀번호 변경 (12자 + 영·숫 + 공백 금지)
4. 변경 완료 후 `/files`로 자동 이동

### 4.2 첫 사용
1. 좌측 사이드바 — "부서" / "팀" / "공유함" 3-section 트리
2. 본인 부서 진입 후 파일/폴더 업로드
3. 파일 클릭 → RightPanel (?file=) — 상세/버전/활동/권한 4탭
4. `?` 키로 단축키 cheat sheet 확인

## 5. 변경 이력 (출시 시점 master 기준)

본 베타는 다음 트랙들의 누적 산출물 (출시 시점 BETA-RELEASE.md `Source` 참조):

- **인증/세션** — Spring Session JDBC + 12자 PW + 강제 변경 + 셀프 가입 (first-user ADMIN) + forgot rate-limit + email async
- **권한 모델** — UUID-keyed permissions + `IbizDrivePermissionEvaluator` + RBAC role 3종 (MEMBER/AUDITOR/ADMIN)
- **파일/폴더 도메인** — V1~V19 마이그레이션 + soft-delete + hard purge cron + 버전 + 공유 + 휴지통
- **팀 도메인** — team-centric-pivot Plan A/A2 (생성/멤버/리더/archive) + Plan C share subject_type='team'
- **관리자 콘솔** — 11 sub-tab + audit export(CSV/JSON) + cron toggle + quota mutation 5-phase
- **디자인 fidelity** — 디자인 zip 1:1 매핑 (메인 탐색기 + Admin Console + AdminMembers + 파일 타입 아이콘)
- **2인 승인 framework** — `pending_admin_approvals` + role_change/retention_change handler (Phase 1~3)
