## 리스크

- **TopBar 테스트 router mocking**: 기존 TopBar 테스트는 next/navigation 의존 없음. URL sync 도입 시 router.replace mock 필요.
- **검색 모드 StatusBar 처리**: 단순히 hide 결정 (위 7.1).
- **선택 + 폴더 이동(검색 결과 안 폴더 클릭)**: selection은 폴더 변경 시 clear (FileTable의 `useEffect` `[folderId, clear]`). SearchResults에는 동일 효과 없음 — 검색 결과 내 다른 폴더로 router.push 시 ClientFilesPage 재마운트되며 새 folderId 반영, FileTable의 clear 가 동작. ✅
